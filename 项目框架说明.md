## 三、数据库设计

数据库名称：`digital_companion`（MongoDB）

### 3.1 users 集合 — 用户信息

| 字段 | 类型 | 说明 | 索引 |
|------|------|------|------|
| `_id` | ObjectId | 主键 | 自动 |
| `username` | String | 用户名 | 唯一索引 `idx_users_username` |
| `email` | String | 邮箱 | 唯一 |
| `password_hash` | String | bcrypt 加密密码 | — |
| `nickname` | String | 昵称 | — |
| `created_at` | DateTime | 注册时间 | 降序索引 `idx_users_created_at` |
| `updated_at` | DateTime | 更新时间 | — |

### 3.2 digital_persons 集合 — 数字人配置

| 字段 | 类型 | 说明 |
|------|------|------|
| `_id` | ObjectId | 主键 |
| `user_id` | ObjectId | 关联用户（索引 `idx_dp_user_id`） |
| `name` | String | 数字人姓名 |
| `relationship` | String | 与用户的关系（如"奶奶""朋友"） |
| `avatar` | String | 头像（Base64 或静态路径） |
| `personality_traits` | Array[String] | 性格特征列表 |
| `background_story` | String | 背景故事 |
| `speaking_style` | String | 说话风格 |
| `user_description` | String | 关于用户的补充描述 |
| `experiences` | Array[Object] | 人生经历时间线（{year, event}） |
| `tts_config` | Object | TTS 语音配置 |
| `speaking_video_url` | String | 动态视频 URL（阿里云 OSS） |
| `last_avatar_for_video` | String | 视频生成时使用的头像快照 |
| `created_at` / `updated_at` | DateTime | 时间戳 |

**索引**：`idx_dp_user_id`（user_id 升序），`idx_dp_user_created`（user_id + created_at 联合索引）

### 3.3 conversations 集合 — 对话记录元数据

| 字段 | 类型 | 说明 |
|------|------|------|
| `_id` | ObjectId | 主键 |
| `user_id` | ObjectId | 关联用户 |
| `digital_person_id` | ObjectId | 关联数字人 |
| `title` | String | 对话标题 |
| `message_count` | Integer | 消息总数 |
| `summary` | String | 历史对话摘要（超长时自动压缩） |
| `last_message_at` | DateTime | 最后消息时间 |
| `created_at` / `updated_at` | DateTime | 时间戳 |

**索引**：`idx_conv_user_updated`（user_id + updated_at），`idx_conv_dp_id`（digital_person_id）

### 3.4 messages 集合 — 消息详情

| 字段 | 类型 | 说明 |
|------|------|------|
| `_id` | ObjectId | 主键 |
| `conversation_id` | ObjectId | 关联对话 |
| `role` | String | 消息角色（"user" / "assistant"） |
| `content` | String | 消息内容 |
| `emotion` | String | 细粒度情感标签（10 种） |
| `emotion_polarity` | String | 情感极性（positive/negative/neutral） |
| `timestamp` | DateTime | 时间戳 |

**索引**：`idx_msg_conv_time`（conversation_id + timestamp 联合索引）

### 3.5 user_memories 集合 — 记忆存储

| 字段 | 类型 | 说明 |
|------|------|------|
| `_id` | ObjectId | 主键 |
| `user_id` | ObjectId | 关联用户 |
| `digital_person_id` | ObjectId | 关联数字人 |
| `source_conversation_id` | ObjectId | 来源对话 |
| `category` | String | 分类（career/emotion/personal/goal/event） |
| `content` | String | 记忆内容 |

**索引**：`idx_mem_user_dp`（user_id + digital_person_id），`idx_mem_hash`（content_hash），`idx_mem_user_category`（user_id + category）

### 3.6 user_profiles 集合 — 用户职业档案

| 字段 | 类型 | 说明 |

## 五、核心功能模块说明

### 5.1 多场景动态路由（prompt_service.py）

系统根据用户输入自动识别 5 类对话场景：

| 优先级 | 场景 | 触发关键词示例 | 响应策略 |
|--------|------|----------------|----------|
| 1（最高） | 危机干预 | "不想活""想死""自杀""绝望" | 温柔安抚 + 强制推送心理援助热线 |
| 2 | 面试辅导 | "面试""笔试""offer""HR面" | STAR法则 + 专业知识库 + 天气提醒 |
| 3 | 情感支持 | "焦虑""难过""崩溃""想哭" | 共情倾听 + 情绪建议 |
| 4 | 职业规划 | "找工作""秋招""简历""实习" | 职业分析 + 岗位推荐 |
| 5（默认） | 日常闲聊 | 其他内容 | 自然陪伴对话 |

### 5.2 智能记忆管理（memory_service.py）

- **自动提取**：每 2 条消息触发一次 AI 记忆提取
- **五类分类**：career / emotion / personal / goal / event
- **双重去重**：SHA256 哈希精确去重 + Jaccard 相似度语义去重（阈值 0.85）
- **向量检索**：text-embedding-v3 向量化 → 余弦相似度 Top5 召回
- **加权评分**：`综合得分 = 相似度 × 0.7 + 重要性 × 0.3`
- **摘要压缩**：消息超 20 条时自动生成历史摘要

### 5.3 提示词工程体系（prompt_service.py）

System Prompt 采用模块化组装策略：

```
身份声明 → 安全规则（最高优先级）→ 基础人设 → 经历时间线 →
用户档案 → 场景指令（动态注入）→ 天气情报 → 面试知识 →
对话原则 → 历史摘要 → 长期记忆（Top15）→ 短期上下文
```

### 5.4 AI 对话服务（ai_service.py）

- **同步模式**：带 3 次重试机制，递增等待间隔
- **流式模式**：async generator 逐块 yield + 45 秒 watchdog 超时保护
- **输出过滤**：正则黑名单过滤危险引导词
- **记忆提取**：temperature=0.1 确保 JSON 格式稳定

### 5.5 并发控制与安全

- **并发限流**：asyncio.Semaphore 用户级限流（每用户 2 并发）
- **认证安全**：JWT HS256 签名（7 天有效期）+ bcrypt 密码加密
- **输入安全**：XSS 清理 + 消息长度 1~2000 字符限制
- **输出安全**：AI 回复正则黑名单过滤

## 六、启动方式

### 6.1 后端启动

```bash
cd backend
pip install -r requirements.txt     # 安装 Python 依赖
python run.py                        # 启动后端服务（默认 http://localhost:8000）
```

**前置要求**：
- Python 3.11+
- MongoDB 7.x 运行在 `localhost:27017`
- `.env` 文件配置 AI_API_KEY 等环境变量

### 6.2 前端启动

```bash
cd frontend
npm install                          # 安装前端依赖
npm run dev                          # 启动开发服务器（默认 http://localhost:5173）
```

**前置要求**：
- Node.js 18+
- Vite 自动将 `/api` 请求代理到后端 `localhost:8000`

### 6.3 外部服务依赖

| 服务 | 用途 | 是否必须 |
|------|------|----------|
| MongoDB | 数据存储 | ✅ 必须 |
| 阿里云 DashScope API | AI 对话 / 向量化 / 语音识别 | ✅ 必须 |
| GPT-SoVITS 本地服务 | 语音合成 | ❌ 可选（不启动则无语音输出） |
| 高德开放平台 API | 天气查询 | ❌ 可选（不配置则无天气提醒） |
