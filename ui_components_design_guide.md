# 数字人陪伴系统 - 页面与组件清单

根据后端已实现的功能接口，前端需要设计以下页面和具体组件：

## 1. 登录页 (Login Page)
**核心页面元素**：
- 系统 Logo / 标题
- 账号/用户名输入框
- 密码输入框
- 登录按钮
- 全局通知组件 (Toast)：用于显示登录成功或失败错误信息

## 2. 首页 (Home Page - 数字人选择区)
**核心页面元素**：
- 顶部导航栏 / 个人信息区 (显示当前登录用户)
- 退出登录按钮
- 数字人卡片列表 (Grid/List)
  - **单张数字人卡片组件**：
    - 角色头像/形象图
    - 角色名称
    - 角色简介说明
    - 选中/进入聊天按钮

## 3. 聊天页 (Chat Page)
**核心页面元素 (分为侧边栏、顶部栏、聊天主区域和底部输入区)**：

### 3.1 侧边栏 (Sidebar) 
- 新建对话按钮
- 历史会话列表组件 (支持点击切换历史记录)
- 当前用户信息展示

### 3.2 顶部状态栏 (Header)
- 当前对话的数字人基本信息展示 (头像、名称)
- **特定场景徽章组件 (Badge/Tag)**：当后端检测到处于特殊场景 (如: 💼 面试模式) 时显示的醒目标签。

### 3.3 聊天主内容区 (Message Area)
- **用户消息气泡组件**
- **AI 消息气泡组件** (需支持 Markdown 格式文本渲染)
- **加载/打字指示器组件 (Loading Indicator)**：AI 正在回复时的等待动效
- **分界线组件**：用于分割时间线或新旧对话上下文
- **【核心场景级组件】**：
  - **面试小贴士面板 (Interview Tips Panel)**：列表型组件，展示后端的面试建议数组 (如"提前10分钟到", "STAR法则"等)。需支持展开/折叠。
  - **天气信息卡片 (Weather Card)**：展示特定城市的温度、天气描述、风力及穿衣/带伞建议。

### 3.4 底部输入区 (Input Area)
- 文本多行输入框 (Textarea)
- 发送按钮
- **阻断/取消按钮 (Stop Button)**：在 AI 流式输出期间，发送按钮需变为取消按钮，用于手动打断回复。

## 4. 全局特殊功能组件
- **危机干预弹窗 (Crisis Modal)**：
  - 触发条件：系统检测到极端消极情绪。
  - 组件内容：包含援助热线列表 (北京热线、全国热线) 、具有安抚性的文字说明。此弹窗应为全局覆盖式的模态框 (Modal)。
