## 摘要 (Abstract)

### 中文摘要
**背景**：随着人工智能和虚拟现实技术的发展，基于大模型的陪伴型情感对话机器人成为缓解现代人精神压力的新途径。
**工作说明**：本文设计并实现了一款名为“在”的数字人情感陪伴系统。系统采用前后端分离架构，前端应用现代化的 Vue 3 框架和 GSAP 动画构建深度玻璃拟物化（Glassmorphism）响应式 UI；后端基于高性能的 FastAPI 框架构建，利用 MongoDB 进行高弹性数据存储。
**创新点/核心技术**：
1. 集成了通义千问大模型进行核心自然语言生成。
2. 通过双重记忆管理架构（短期上下文滑动窗口 + 长期向量检索识别），基于 `text-embedding-v3` 实现了精准的记忆寻回，增强了多轮对话的一致性。
3. 构建了“动态多场景路由”机制，涵盖日常闲聊、情感支持、职业辅导、危机干预等特定场景自动切换。
4. 融合了开源的 GPT-SoVITS 语音克隆技术与多模态视频生成服务（WAN2图生视频），实现高逼真度的视听交互体验。
**结论**：测试表明，该系统能在高并发场景下保持低延迟，并有效拦截危险信息，为用户提供沉浸式、个性化的情感陪伴体验。

**关键词**：数字人；大语言模型；情感陪伴；FastAPI；Vue 3；向量检索

## 1. 绪论

### 1.1 研究背景与意义
- 现代社会节奏加快，孤独感和心理亚健康问题频发。
- 大模型（LLM）突破了传统对话系统“死板”的局限，使其具有强大的逻辑推理与共情能力。
- 设计一款懂记忆、能共情，并提供丰富视觉反馈的“数字人陪伴系统”极具社会价值和技术应用双重意义。

### 1.2 国内外研究现状
- **传统聊天机器人**：如 ELIZA、Siri，缺乏深度上下文理解。
- **现代大模型大火**：ChatGPT、Claude 展示了AGI雏形。
- **个性化数字人领域**：目前市面大多年费昂贵，或者记忆功能割裂，缺乏一个多模态（文字、语音、微表情视频）集成的端到端个人部署方案。

### 1.3 本文主要研究内容与贡献
1. 提出了一套完整的记忆摘要与检索算法解决大模型“金鱼记忆”痛点。
2. 重构并设计了工业级的深色拟物态（Deep Dark Glassmorphism）前端交互界面。
3. 实现了高阶系统的安全性，采用正则表达式黑名单和危机拦截。

## 2. 核心相关技术介绍

### 2.1 高级前端工程化：Vue 3 与 GSAP 
- 介绍 Vue 3 的 Composition API 带来的状态管理提升 (Pinia)。
- 介绍 GSAP（GreenSock Animation Platform）如何实现前端组件复杂的交错入场动画和视觉过度，增强沉浸感。

### 2.2 异步全栈骨架：FastAPI 与 Uvicorn
- 介绍 FastAPI 基于 Starlette 和 Pydantic 的异步协程优势。

### 2.3 动态文档数据库：MongoDB 与 Motor
- 介绍 MongoDB 的 schema-free 特性完美适配非结构化的“聊天记录”与“用户记忆”存储需求。

### 2.4 大规模语言生成与多模态大模型
- **通义千问 (Qwen-Plus)**：充当系统的大脑。
- **text-embedding-v3**：提供记忆句向量化引擎，实现语义相近即可检索。
- **WAN2 图生视频 / GPT-SoVITS**：提供肢体行为和语音合成的支持。

### 2.5 服务器单向事件流技术（SSE）
- 规避了传统 HTTP 轮询的延迟，实现长文本的打字机（流式）极速响应。

## 3. 系统需求分析

### 3.1 可行性分析
- 技术可行性：前后端技术栈成熟，大模型 API 开放稳定。
- 经济/操作可行性：本地部署降低隐私泄露风险。

### 3.2 功能性需求分析
（可用 Visio/Drawio 画出用例图）
1. **用户认证模块**：账户注册与 JWT Token 安全登录。
2. **数字人管理模块**：数字人的创建、人格设定、关联 TTS 本地映射。
3. **沉浸式交互模块**：基于 SSE 的流式问答，动态背景响应。
4. **长效记忆模块**：系统后台静默抽认记忆（每两轮自动摘要提取），动态干预当前 Prompt。
5. **多模态扩展分析模块**：视频对话生成、天气情报对接。

### 3.3 非功能性需求分析
- 性能需求：并发限制，单用户最多开2条并发，大模型接口设置超时（如 45 秒 watchdog 防止阻塞）。
- 安全需求：敏感词过滤，高危话题强制熔断/安抚（如轻生等言论触发特定响应策略）。界面安全性防注入。

## 4. 系统设计

### 4.1 总体架构设计
（建议画图：三层架构 -> 视图表现层、应用服务层、数据存储层）
后端服务化层（Service Layer）解耦设计：`ai_service.py`、`memory_service.py`、`prompt_service.py`。

### 4.2 核心业务流程设计
1. **多级过滤与响应流程**：用户输入 -> 安全拦截判断 -> 历史记忆检索 -> 组装 System Prompt -> 调用千问 API -> 流式返回 -> 返回后后台生成记忆。
2. **向量记忆存取流程**：内容提取分类（career/emotion等）-> SHA256防重验证 -> 生成嵌入向量 -> 存入 MongoDB。

### 4.3 数据库设计 (MongoDB Schema)
（可使用 E-R 图或者集合关系图描述）
由于我们是不需要建立严格外键关联的 NoSQL 数据库，主要详细列出上一步生成的《项目框架说明.md》中的以下字段：
- `users` 集合：用于身份基础验证。
- `digital_persons` 集合：包含身份、关系、音色、性格列表等。
- `conversations` 与 `messages` 集合。
- `user_memories` 集合：包含 `content_hash` 精确去重，`embedding` 余弦距离比对字段。

## 7. 总结与展望

### 7.1 本文主要工作总结
重申项目不仅是一个对话框，而且沉淀了自研的长期记忆向量召回架构、安全过滤拦截、多场景态动态适配等企业标准技术手段。同时前端的深度重构满足了良好的商业化视觉标准。

### 7.2 系统存在的不足与未来展望
- 目前声音克隆 (GPT-SoVITS) 较吃本地算力，若网络延迟可能影响沉浸感，未来可加入 WebSocket 全双工微秒级流式语音传输。
- 进一步增加用户的生物特征（使用摄像头分析疲劳度），完善“视觉-情感”双通道陪伴功能。
