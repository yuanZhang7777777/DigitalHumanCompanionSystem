## 4. 多模态交互时序图

```mermaid
sequenceDiagram
    participant U as 用户
    participant FE as 前端
    participant BE as 后端
    participant ASR as ASR服务
    participant V as Vision
    participant LLM as 通义千问
    participant TTS as TTS服务
    participant DB as MongoDB

    alt 语音输入
        U->>FE: 按住语音录制
        FE->>ASR: 上传音频文件
        ASR-->>FE: 返回识别文本
        FE->>FE: 显示识别结果
    end

    alt 图片输入
        U->>FE: 上传图片
        FE->>BE: 发送图片URL + 消息
        BE->>V: VisionService解析
        V-->>BE: 返回图片描述
    end

    U->>FE: 发送消息
    FE->>BE: POST /api/conversations/message

    BE->>DB: 查询用户记忆
    DB-->>BE: 返回相关记忆

    BE->>LLM: 发送对话请求
    LLM-->>BE: SSE流式响应

    loop 流式生成
        BE-->>FE: SSE chunks
        FE->>FE: 逐字渲染消息
    end

    alt 启用语音播报
        BE->>TTS: 请求语音合成
        TTS-->>BE: 音频流
        BE-->>FE: 音频流播放
    end

    alt 启用数字人视频
        BE->>LLM: 生成口型同步视频
        LLM-->>BE: 视频URL
        BE-->>FE: 返回视频URL
        FE->>FE: 播放数字人视频
    end

    BE->>DB: 保存消息记录
    BE->>BE: 触发后台记忆提取
    BE->>BE: 检查是否需要生成摘要

    BE-->>FE: 返回完整响应
```


## 6. 核心代码逻辑分析

### 6.1 多场景路由优先级

| 优先级 | 场景 | 关键词示例 |
|--------|------|------------|
| 1 | 危机干预 | 不想活/想死/轻生/自杀/抑郁症/绝望 |
| 2 | JD深度分析 | JD/岗位描述/招聘要求 |
| 3 | 面试辅导 | 面试/offer/STAR法则/群面/笔试 |
| 4 | 情感支持 | 焦虑/迷茫/压力/难过/孤独 |
| 5 | 职业规划 | 职业/工作/就业/职场/跳槽 |
| 6 | 轻松聊天 | 默认 |

### 6.2 记忆提取算法

```
去重双重机制:
1. SHA256 哈希精确匹配
2. Jaccard 相似度检测 (阈值 0.85)

存储向量维度: 1536 (text-embedding-v3)

召回权重计算:
召回分数 = 语义相似度 x 0.7 + 重要性评分 x 0.3
```

### 6.3 并发控制

```
每用户最大并发: 2
限流策略: Semaphore 信号量
超时监控: 45秒 Watchdog
```

### 6.4 提示词组装顺序

```
1. 数字人基础人设 (name, relationship, personality)
2. 场景专属Prompt片段 (SCENE_PROMPTS[scene])
3. 用户职业档案 (profile)
4. 相关记忆上下文 (user_memories)
5. 天气情报 (weather)
6. 历史摘要 (summary)
7. 安全准则 (SAFETY_RULES)
```
