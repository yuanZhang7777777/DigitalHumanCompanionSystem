## 二、 核心亮点写作用词素材（可直接化用）

您的这套系统在大学毕设答辩中，具备以三大“高分护城河”级别的**技术亮点**，在论文中需要浓墨重彩地描述：

### 亮点一：具备“心理分析与动态场景路由”的超强智能大脑
传统大模型往往是冰冷的问答工具。而本系统在接收到用户的语句后，设计了特殊的 **情感与场景分析管线（Pipeline）**：
- **情感计算**：通过前置轻量级节点，计算用户语句中的情绪极度（如高兴、崩溃、焦虑）。
- **重大指令注入（Prompt Injection）与防幻觉**：面对网络知识获取，引入外部第三方实时 API 结构（如高德天气服务），利用极高优先级的后台指令绕过模型原生屏蔽词限制，确保给用户真实可靠的数据支撑。
- **危机干预机制**：若模型识别到自残等词汇，立刻切断常规模板，向前端推送带有颜色预警和求助热线的急救帧数据（SSE），极大地增强了产品安全伦理特性。

### 亮点二：“偷听大脑”与语义空间级（Vector）长期记忆
如何让数字人“记住”用户？相比机械化的数据库存储，本系统实现了**后台记忆窃取智能体**机制：
- 每次用户与数字人通讯结束，系统并不会阻塞当前对话界面，而是使用异步多线程/协程队列，开启后台影子模型（Shadow Model）。
- 该模型分析对话内容，清洗无关寒暄，只针对用户的偏好、职业、目标提取核心 JSON 记录格式，存入 MongoDB。在用户下次上线时，重新提取最近相关度最高的五个记忆节点合并入角色的上下文，实现了数字人的“拥有岁月感”。

### 亮点三：完全现代化的全双工通信感知（SSE）
放弃了传统的轮询方式（Polling），采用了 **Server-Sent Events (SSE)** 流式传输技术：
- 让大模型生成的每一滴字符（Chunk）都像打字机一样灵动地呈现在前端。
- 首字响应时间由传统的 10秒 级别优化至 **平均 500ms**，实现了用户对聊天的即时性心理反馈需求。
- 一方面传递流式回答文字，一方面传递附带元数据的系统特殊事件（比如：JD 职场解析数据结构、气象面板卡片）。
